# 基于 Pretext 的 EPUB 阅读器开发任务文档

## 1. 文档目标

本文档用于将《需求文档》和《技术文档》进一步拆分为可执行开发任务，作为后续实现顺序与验收依据。

约束原则：

- 所有任务合计必须覆盖需求文档的全部范围
- 每个任务必须带测试要求
- 开发顺序默认从上到下执行
- 未完成前置任务，不进入后置任务

关联文档：

- [需求文档](./需求文档.md)
- [技术文档](./技术文档.md)

## 2. 总体执行规则

### 2.1 任务执行规则

- 每个任务完成后，必须先补测试，再进入下一个任务
- 每个任务必须提交可运行代码，不接受只有设计没有实现的完成态
- 每个任务必须更新相应类型定义、文档或示例
- 如果任务影响公开 API，必须同步更新 README 或 API 文档

### 2.2 测试执行规则

- 每个任务至少包含单元测试
- 涉及模块协作的任务必须补集成测试
- 涉及用户交互或浏览器行为的任务必须补端到端测试或 demo 验证步骤
- 测试名称必须直接体现任务目标，避免空泛命名

### 2.3 完成定义

一个任务仅在以下条件都满足时才算完成：

- 代码完成
- 测试完成并通过
- 类型检查通过
- 相关文档更新
